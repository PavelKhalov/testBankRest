# Система управления банковскими картами

REST API для управления банковскими картами пользователей: регистрация и аутентификация, создание и обслуживание карт, переводы между своими картами, административное управление пользователями и картами.

## Технологии

- **Java 21**, **Spring Boot 4.1**
- **Spring Web MVC** - REST-контроллеры
- **Spring Data JPA** + **Hibernate** - доступ к данным
- **Spring Security** + **JWT** (jjwt) - аутентификация и авторизация по ролям
- **PostgreSQL** - хранение данных
- **Liquibase** - версионирование и миграции схемы БД
- **Springdoc OpenAPI (Swagger UI)** - документация API
- **Lombok** - сокращение шаблонного кода
- **JUnit 5 + Mockito** - модульное тестирование
- **Docker / Docker Compose** - контейнеризация приложения и БД

## Роли и доступ

В системе две роли: **USER** и **ADMIN**.

| Область | Доступ |
|---|---|
| `/api/auth/**` | без авторизации (регистрация, логин) |
| `/api/admin/**` | только роль `ADMIN` |
| `/api/**` (остальные) | любой аутентифицированный пользователь (`USER`/`ADMIN`) |

Авторизация - по JWT-токену, который передаётся в заголовке `Authorization: Bearer <token>`. Сессии не хранятся (stateless), токен выдаётся при логине/регистрации и содержит имя пользователя.

## Возможности

### Пользователь (роль USER)

- Регистрация и логин по логину/паролю
- Просмотр списка своих карт (с пагинацией)
- Просмотр карты по id и баланса карты
- Запрос на блокировку своей карты
- Перевод денежных средств между своими картами

### Администратор (роль ADMIN)

- Создание новой карты для пользователя
- Просмотр всех карт в системе (с пагинацией и фильтром по владельцу)
- Блокировка / активация любой карты
- Удаление карты
- Просмотр списка пользователей
- Изменение роли пользователя
- Включение / отключение пользователя (блокировка доступа без удаления)
- Удаление пользователя

## Безопасность данных карты

- Номер карты хранится в базе данных в **зашифрованном** виде (AES, `CardNumberEncryptor`), а не в открытом тексте.
- В ответах API номер карты возвращается в **маскированном** виде — видны только последние 4 цифры (`**** **** **** 1234`).
- Пароли пользователей хешируются перед сохранением (`PasswordEncoder`), в открытом виде не хранятся и не возвращаются.

## Модель данных

**User** - пользователь системы: логин, имя, фамилия, email, пароль (хеш), роль, флаг активности.

**Card** - банковская карта: зашифрованный номер, последние 4 цифры, баланс, владелец, срок действия, статус (`ACTIVE` / `BLOCKED` / `EXPIRED`), дата создания.

Схема БД создаётся автоматически при старте приложения через Liquibase-миграции (`db/changelog`).
